# Data accounting and analysis system
### Система учета и анализа данных

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](https://opensource.org/licenses/MIT)

REST-сервис на **FastAPI** для приема показаний устройств, сохранения их в БД и получения аналитики по значениям `x`, `y`, `z`.

## Описание

Сервис позволяет:

- регистрировать пользователей;
- добавлять устройства пользователям;
- принимать показания устройства по его идентификатору;
- получать аналитику устройства по показаниям:
  - минимальное значение;
  - максимальное значение;
  - количество;
  - сумма;
  - медиана;
- получать аналитику по всем устройствам пользователя.

Данные сохраняются в MySQL.

## Технологии

- Python 3.11
- FastAPI
- SQLAlchemy
- PyMySQL
- MySQL 8
- Docker
- Docker Compose

## Структура БД

### `users`
- `id` — идентификатор пользователя
- `name` — имя пользователя
- `email` — email пользователя

### `devices`
- `id` — идентификатор устройства
- `name` — имя устройства
- `user_id` — ссылка на пользователя

### `device_statuses`
- `id` — идентификатор записи
- `device_id` — ссылка на устройство
- `x` — значение показания
- `y` — значение показания
- `z` — значение показания
- `time` — дата и время получения показаний

## Запуск через Docker

### 1. Создать файл `.env`

Скопируйте `.env.example` в `.env` и при необходимости измените значения:

```env
MYSQL_ROOT_PASSWORD=change_me_root_password
MYSQL_DATABASE=get_device_params
MYSQL_USER=app_user
MYSQL_PASSWORD=change_me_user_password
```

### 2. Собрать образ контейнера

```bash
docker-compose up --build
```

### 3. Открыть сервис
- API: http://localhost:4000
- Swagger: http://localhost:4000/docs
## API

### GET /

Проверка работы сервиса.

### GET /users

Получить список всех пользователей.

### GET /users/{user_id}

Получить информацию о пользователе по ID.

### GET /users/{user_id}/devices

Получить список всех устройств пользователя.

### POST /register

Создать пользователя.

Пример запроса:

{
  "name": "Ivan",
  "email": "ivan@example.com"
}

### POST /device

Добавить новое устройство пользователю.

Пример запроса:

{
  "name": "Device-1",
  "user_id": 1
}

### POST /devices/{device_id}/statistics

Передать показания устройства.

Пример запроса:

{
  "x": 1.2,
  "y": 3.4,
  "z": 5.6
}

### GET /users/{user_id}/devices/{device_id}/statistics/analysis

Получить аналитику по конкретному устройству.

Параметры:

- from_date
- to_date

Пример:

/users/1/devices/2/statistics/analysis?from_date=2025-01-01T00:00:00&to_date=2025-01-31T23:59:59

### GET /users/{user_id}/statistics/analysis

Получить аналитику по всем устройствам пользователя.

Параметры:

- from_date
- to_date

Пример:

/users/1/statistics/analysis?from_date=2025-01-01T00:00:00&to_date=2025-01-31T23:59:59

## Примечания

- При первом запуске создаются таблицы из файла init.sql.
- Данные сохраняются в MySQL.
- Аналитика строится по данным, уже записанным в БД.
- Для работы через Docker сервис БД доступен по имени db.
